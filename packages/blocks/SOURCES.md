# @domphy/blocks — Sources

**Methodology: clean-room implementation.** Every component in this package was implemented independently from a written functional/visual specification produced by a separate research pass — that research pass never saw or wrote any implementation code, and the implementation pass never browsed, fetched, or otherwise viewed the original third-party website, docs page, or source repository. No third-party source code was copied, transcribed, or redistributed anywhere in this package.

Reference URLs below are kept for two reasons only: (1) attribution/transparency about which publicly documented UI pattern each component reimplements, and (2) as the comparison target for the on-demand Playwright visual-compare script (`pnpm --filter @domphy/blocks visual-compare`), which screenshots the live reference and the local implementation side by side for manual QA. Reference screenshots are never committed to this repository.

shadcn/ui and Magic UI free/open-source components are MIT-licensed; this package does not rely on that license to justify inclusion, since nothing was copied from them in the first place — the same clean-room process was applied uniformly to both sources.

Status legend: `ported` — implemented to the full recorded spec. `partial` — implemented with a noted, honest gap (see Notes). `skipped` — spec could not be reasonably implemented; not exported from the package.

All 173 components were also subjected to an automated visual-diff QA pass (screenshotting the local render next to the original reference) — ~35 real rendering bugs were found this way and fixed (see git history for details); every entry below reflects the POST-fix state.

Total components: 173 (shadcn/ui blocks + chart recipes: 97, Magic UI: 76).

Aceternity UI was removed entirely (2026-07-05): those 79 components had never been diffed against Aceternity's real source (no clean public repo existed at port time), and a direct source-diff pass on a couple of them (`pointerHighlight`, and cross-checking `bentoGrid` against Magic UI's real source) turned up genuine, not-approximation-level fidelity gaps — wrong animation technique entirely, not a styling nuance. Rather than re-diff all 79 against Aceternity's registry JSON one at a time, the whole category was dropped.

**Full direct-source-diff pass (2026-07-05):** following that discovery, all remaining 173 shadcn/ui + Magic UI components were individually diffed line-by-line against the real cloned upstream repos (`shadcn-ui/ui`, `magicuidesign/magicui` — both MIT-licensed, so reading their source for verification carries no license concern; only the original clean-room *build* process avoided it). This found and fixed 58 real, containable fidelity gaps — wrong animation directions, missing gridlines/legends/stacking on chart recipes, inverted keyframe/timing values, missing form elements, wrong default prop values, and one component (`rainbowButton`) whose entire default visual identity had been invented rather than matched. 4 further gaps were initially left flagged pending a human call (`fileTree`, `codeComparison`, `text3dFlip`, `iconCloud`); all four were subsequently implemented in a follow-up pass (see their own entries below). Every fixed entry below carries a "Direct-source-diff fix (2026-07-05)" note appended to its original fidelityNotes.
