\begin{abstract}
Current web user interface (UI) architectures are predominantly structured around hierarchical component recursion, which introduces structural nesting and reconciliation overhead. We present \textbf{Domphy}, a reactive framework based on the \textbf{Compositional Atomic Reactive Mapping (CARM)} model. CARM defines UI elements as flat compositional objects formed through a deterministic merge operator, separating structural composition from reactive evaluation. Instead of reconstructing virtual trees upon state changes, Domphy performs a single-pass structural reification that yields persistent runtime entities with property-level reactive bindings. 

Under bounded dependency assumptions, reactive updates are executed in constant time ($O(1)$) per affected property, avoiding global tree traversal and diff-based reconciliation. Empirical evaluation demonstrates reduced structural depth, simplified source definitions, and competitive runtime performance compared to representative component-based frameworks. The results suggest that flat compositional semantics combined with atomic reactive mapping provide a viable alternative computational model for declarative UI construction.
\end{abstract}
